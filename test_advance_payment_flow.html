💰 Advance Payment Flow Test

🔄 Complete Advance Payment Flow:

1. POS Cart Entry

  • Customer selects items in POS cart
  • Customer type: "Customer" (not "Counter")
  • Advance Payment: Enter amount in "Amount Paid Now" field
  • Example: Total = 500 DH, Advance = 200 DH

2. Sale Processing

  • API receives: advance_payment: 200.00
  • Database saves: sales.advance_payment = 200.00
  • Payment recorded: customer_payments table with advance_payment type
  • Remaining calculated: 500 - 200 = 300 DH

3. Customer Details View

  • Transaction shows: Total: 500 DH, Advance: 200 DH, Remaining: 300 DH
  • Status: "Remaining" (red badge)
  • Action button: "Pay Rest" appears for unpaid balance

4. Pay Rest Functionality

  • Click "Pay Rest": Opens payment modal
  • Modal shows: Remaining amount (300 DH)
  • Payment modes: All available payment methods
  • Process payment: Updates sales.advance_payment to 500 DH
  • Creates record: customer_payments with remaining_payment type
  • Status changes: "Paid" (green badge)

📊 Database Tables Updated:

Table Field Value
sales advance_payment 200.00 (initial) → 500.00 (after pay rest)
customer_payments payment_type 'advance_payment' → 'remaining_payment'
customer_payments payment_amount 200.00 → 300.00
