🔍 Root Cause:

The api/customers/pay_remaining.php was trying to update an updated_at column that doesn't exist in the sales table.

  3. Simplified the query to only update the advance_payment field

📝 Before Fix:

UPDATE sales SET advance_payment = ? WHERE id = ?

The SQL file now includes:

  • This ensures the column exists for future functionality
  • Current API works without the updated_at field
